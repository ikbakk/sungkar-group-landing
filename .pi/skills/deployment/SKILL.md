# Deployment & DevOps Skill

Manage build configuration, deployment, and DevOps.

## Build Configuration (`astro.config.mjs`)

```javascript
import { defineConfig } from "astro/config";
import tailwindcss from "@tailwindcss/vite";
import robotsTxt from "astro-robots-txt";
import react from "@astrojs/react";
import sitemap from "@astrojs/sitemap";
import mdx from "@astrojs/mdx";
import { SITE_URL } from "./src/lib/site-config";

export default defineConfig({
  site: SITE_URL,                // "https://www.sungkargroup.com"
  vite: {
    plugins: [tailwindcss()],    // Tailwind v4 Vite plugin
  },
  i18n: {
    defaultLocale: "id",
    locales: ["id", "en", "ar", "ms", "zh"],
    routing: {
      prefixDefaultLocale: false, // ID locale → root URLs
      redirectToDefaultLocale: false,
    },
  },
  integrations: [
    react(),
    sitemap({
      i18n: {
        defaultLocale: "id",
        locales: {
          id: "id-ID",
          en: "en-US",
          ar: "ar-SA",
          ms: "ms-MY",
          zh: "zh-CN",
        },
      },
    }),
    robotsTxt(),
    mdx(),
  ],
});
```

## Key Config Details

### Site URL

The canonical site URL is defined in `src/lib/site-config.ts`:
```typescript
export const SITE_URL = "https://www.sungkargroup.com";
```

This is used by:
- `astro.config.mjs` (`site` field)
- All JSON-LD schema generators (`src/lib/schemas.ts`)
- OG metadata (`src/lib/og-metadata.ts`)
- Sitemap generation
- Canonical URLs

**To change the production domain**, update ONLY `SITE_URL` in `site-config.ts`. All other sources derive from it.

### i18n Configuration

Astro's i18n config handles:
- `defaultLocale: "id"` — The default language
- `locales` — All 5 supported locales
- `prefixDefaultLocale: false` — ID pages at root (`/tentang-kami`, not `/id/tentang-kami`)
- `redirectToDefaultLocale: false` — No automatic redirect to default locale

**Note:** This project uses CUSTOM i18n routing (via `PATH_MAP` in `localize.ts`), not Astro's built-in i18n middleware. The Astro i18n config is primarily for sitemap generation and locale metadata.

### Sitemap Integration

The sitemap integration auto-generates per-locale entries:
```javascript
sitemap({
  i18n: {
    defaultLocale: "id",
    locales: {
      id: "id-ID",
      en: "en-US",
      ar: "ar-SA",
      ms: "ms-MY",
      zh: "zh-CN",
    },
  },
})
```

Each page gets a `<loc>` for each locale with the appropriate `hreflang` annotation.

### Robots.txt

Generated by `astro-robots-txt` integration. Default config allows all crawlers. To customize:

```javascript
robotsTxt({
  policy: [
    { userAgent: "*", allow: "/" },
    { userAgent: "GPTBot", disallow: "/" },  // Block AI crawlers
  ],
})
```

## Adding an Integration

### Step 1: Install the package

```bash
npm install @astrojs/{integration-name}
```

### Step 2: Add to `astro.config.mjs`

```javascript
import integrationName from "@astrojs/{integration-name}";

export default defineConfig({
  integrations: [
    // ... existing
    integrationName({
      // options
    }),
  ],
});
```

### Step 3: Update `src/env.d.ts` if needed

Some integrations require type declarations.

### Step 4: Build test

```bash
npm run build
npm run preview    # Serve the build locally
```

## Build Commands

```bash
npm run build       # Production build → dist/
npm run preview     # Preview the production build locally :4321
npm run dev         # Development server :4321
```

### Build Output

Output goes to `dist/` directory. Build is static (SSG) — no server required.

### Build Verification

After `npm run build`:
1. `npm run preview` — Verify all pages render correctly
2. Check `dist/sitemap-index.xml` — All locale entries present
3. Check `dist/robots.txt` — Correct directives
4. Verify Arabic pages have `dir="rtl"` in HTML
5. Check 404 page works

## Deployment Platforms

The project is currently set up for static hosting. Common deployment targets:

### Netlify (Primary)

The project deploys to Netlify. Config is in `netlify.toml`:
```toml
[build]
  command = "npm run build"
  publish = "dist"

[[headers]]
  for = "/_astro/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "/assets/*"
  [headers.values]
    Cache-Control = "public, max-age=31536000, immutable"

[[headers]]
  for = "/*"
  [headers.values]
    X-Content-Type-Options = "nosniff"
    X-Frame-Options = "DENY"
    Referrer-Policy = "strict-origin-when-cross-origin"
```

1. Connect the GitHub repo to Netlify (or use the GitHub Actions workflow)
2. Set build command: `npm run build`
3. Publish directory: `dist/`
4. Set `NETLIFY_AUTH_TOKEN` and `NETLIFY_SITE_ID` in GitHub secrets for CI

If SSR is needed in the future, add `@astrojs/netlify`:
```bash
npm install @astrojs/netlify
```
And update `astro.config.mjs`:
```javascript
import netlify from "@astrojs/netlify";

export default defineConfig({
  output: "server",
  adapter: netlify(),
});
```

### Cloudflare Pages

Add `@astrojs/cloudflare`:
```bash
npm install @astrojs/cloudflare
```

## Environment Variables

Currently not used (fully static site). If adding:
1. Create `.env` files following the existing `.env.example`
2. Use `import.meta.env.VAR_NAME` in Astro/React code
3. Never commit `.env` files

## CDN Image Migration (Future)

When migrating to an image CDN:
1. Add `CDN_ORIGIN` to `src/lib/site-config.ts`
2. Update `src/lib/schemas.ts` to use CDN_ORIGIN for image URLs in schemas
3. Remote images use string URLs (not barrel imports)
4. Update `scripts/check-images.mjs` to validate remote URLs
