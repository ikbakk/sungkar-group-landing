2. 🔍 Google Search Console

Priority: P0 — Critical. Required for indexing visibility, crawl error monitoring, and performance data.

Add property to Google Search Console TODO
  1. Go to search.google.com/search-console
  2. Sign in with your Google account (use the business/organization account)
  3. Choose URL prefix method and enter: https://www.sungkargroup.com
  4. Verify ownership using one of these methods:

Recommended: DNS TXT Record (Netlify)

  • Select DNS TXT Record verification method
  • Copy the TXT record value provided by Google
  • In your Netlify DNS settings, add a new TXT record for @ or www with that value
  • Wait for DNS propagation (5–30 minutes), then click Verify

Alternatives

  • HTML file upload — Place the Google-provided googleXXXX.html in public/ and deploy
  • HTML tag — Add the meta tag to <head> via Netlify snippet injection or Astro layout
  • Google Analytics — If GA4 is already set up on the site, use the same account
Important: If you use the non-www domain (sungkargroup.com), also add it as a separate property. The 301 redirect means www is the canonical — add both to get full data.
Submit sitemap to Google TODO
  1. In Search Console, go to Sitemaps
  2. Enter sitemap URL: https://www.sungkargroup.com/sitemap-index.xml
  3. Click Submit
  4. Monitor — should show Success with 483 discovered URLs
Expected: All 483 URLs should be indexed within a few days. Check Pages → Indexing after 48–72 hours.
Configure crawl settings TODO
  • Settings → Crawl rate: Leave at "Let Google optimize"
  • Settings → URL parameters: Review if any query params create duplicates (unlikely with Astro SSG)
  • International targeting: Verify hreflang tags are correct
Monitor Core Web Vitals TODO

After verification, check Core Web Vitals in Search Console:

  • Go to Experience → Core Web Vitals
  • Review mobile and desktop scores
  • Key metrics: LCP < 2.5s, FID < 100ms, CLS < 0.1
  • Use PageSpeed Insights for detailed diagnostics
